# 电子档案管理系统 - 前端

## 项目介绍

电子档案管理系统前端是基于Vue3和Element Plus构建的现代化企业级Web应用，提供直观易用的界面，实现档案的全生命周期管理，包括采集、挂接、校验、查询、审批、审计等核心功能。

## 技术栈

- **开发框架**：Vue 3.3.4
- **UI组件库**：Element Plus 2.4.2
- **路由管理**：Vue Router 4.2.5
- **状态管理**：Pinia 2.1.7
- **HTTP客户端**：Axios 1.6.2
- **构建工具**：Vite 5.0.0
- **图表库**：ECharts 6.0.0
- **图标库**：@element-plus/icons-vue 2.3.1

## 系统特性

- ✅ 响应式设计，支持多端访问
- ✅ 现代化UI界面，操作简洁直观
- ✅ 完整的档案管理功能模块
- ✅ 灵活的权限控制和菜单管理
- ✅ 丰富的数据可视化展示
- ✅ 支持大文件分片上传
- ✅ 在线预览多种文件格式
- ✅ 完善的消息通知机制

## 功能模块

### 1. 登录与认证
- 用户登录
- Token认证
- 权限验证
- 记住密码

### 2. 首页与工作台
- 数据概览
- 关键指标统计
- 快捷操作入口
- 待办事项提醒

### 3. 档案采集模块
- **手动上传采集**：单文件手动上传，支持多种文件格式
- **批量上传采集**：多文件批量上传，提高采集效率
- **自动接口采集**：通过配置的外部系统接口自动采集档案数据
- **外部导入采集**：从外部系统批量导入档案数据
- **大文件分片上传**：支持大文件断点续传，确保上传稳定性
- **上传进度显示**：实时显示文件上传进度

### 4. 档案查询模块
- **多条件查询**：支持按档案ID、文件名、档案类型、业务单号等多维度查询
- **分页查询**：高效的分页查询，提升大数据量下的查询性能
- **档案预览**：在线预览档案文件，支持PDF、图片、Office文档等
- **档案下载**：安全的档案文件下载功能
- **档案详情**：查看档案的完整元数据信息
- **高级搜索**：支持复杂查询条件组合

### 5. 档案挂接管理模块
- **手动挂接**：手动建立档案与目标系统的关联关系
- **批量挂接**：批量建立多个档案的挂接关系
- **组合挂接**：支持档案组合挂接，满足复杂业务场景
- **挂接查询**：灵活查询档案的挂接状态和关联关系
- **解除挂接**：解除档案与目标系统的关联
- **挂接重试**：对失败的挂接操作进行重试
- **挂接日志**：查看挂接操作的详细日志

### 6. 档案校验管理模块
- **完整性校验**：检查档案数据的完整性，确保档案内容齐全
- **一致性校验**：验证档案数据与元数据的一致性
- **合规性校验**：确保档案符合管理规范和业务要求
- **校验报告**：生成详细的校验报告，便于问题追踪
- **分类一致性校验**：校验档案分类与文件类型的匹配关系
- **批量校验**：支持批量档案校验

### 7. 用户与权限管理模块
- **用户管理**：用户的增删改查、状态管理、密码重置
- **角色管理**：角色定义、权限分配、状态管理
- **权限管理**：基于RBAC的细粒度权限控制
- **菜单管理**：系统菜单配置与管理
- **用户角色分配**：为用户分配多个角色

### 8. 审批管理模块
- **审批申请**：发起档案相关审批流程
- **审批记录**：完整的审批历史记录
- **审批流程**：支持多级审批流程
- **审批查询**：灵活查询审批记录
- **我的待办**：查看待审批事项
- **我的已办**：查看已审批事项

### 9. 审计与日志模块
- **挂接日志**：记录所有档案挂接操作
- **采集日志**：记录档案采集过程的详细信息
- **系统日志**：记录系统运行情况和关键操作
- **审计报表**：生成操作审计、用户操作、系统性能等审计报表
- **报表生成**：支持按日、周、月、季度、年度生成统计报表
- **日志查询**：灵活的日志查询和筛选功能

### 10. 系统配置模块
- **接口配置**：配置外部系统接口信息、参数映射、密钥等
- **通知管理**：系统通知的发布与管理
- **API监控**：实时监控API接口的调用情况和性能指标
- **档案关系可视化**：图形化展示档案之间的关联关系

## 快速开始

### 环境要求
- **Node.js**：16+
- **npm**：7+ 或 **yarn**：1.22+

### 安装依赖

```bash
npm install
# 或
yarn install
```

### 开发环境运行

```bash
npm run dev
# 或
yarn dev
```

应用将在 http://localhost:3000 启动。

### 生产环境构建

```bash
npm run build
# 或
yarn build
```

构建产物将输出到 `dist` 目录。

### 预览生产构建

```bash
npm run preview
# 或
yarn preview
```

## 项目配置

### 环境变量

项目使用环境变量配置：

```bash
# 后端API基础URL
VITE_API_BASE_URL=http://localhost:8080/archive
```

### 路由配置

路由配置文件位于 `src/router/index.js`，定义了系统的所有页面路由和权限控制。

### API配置

API请求配置文件位于 `src/api/request.js`，使用Axios统一管理API请求，包含请求拦截器和响应拦截器。

## 代码规范

- 使用ESLint进行代码质量检查
- 使用Prettier进行代码格式化
- 遵循Vue 3 Composition API最佳实践
- 组件命名采用PascalCase
- 路由命名采用kebab-case
- 文件命名采用kebab-case

## 开发指南

### 添加新页面

1. 在 `src/views/` 目录下创建新的页面组件
2. 在 `src/router/index.js` 中配置路由
3. 在 `src/api/` 中添加相应的API请求
4. 在 `src/store/` 中添加状态管理（如有需要）
5. 在菜单配置中添加新菜单项

### 添加新组件

1. 在 `src/components/` 目录下创建新组件
2. 使用Vue 3 Composition API编写组件逻辑
3. 确保组件的可复用性和可维护性
4. 添加组件文档和示例

### 文件上传说明

系统支持大文件分片上传，相关逻辑在：
- API：`src/api/collection.js`
- 组件：可复用的上传组件

### 文件预览说明

系统支持多种文件格式的在线预览：
- PDF文档
- 图片文件（JPG、PNG、GIF等）
- Office文档（Word、Excel、PowerPoint）
- 文本文件
